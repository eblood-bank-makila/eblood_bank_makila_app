┌───────────────────────────────────────────────────────────────────────────┐
│  PHASE 1: USER INTERACTION (Flutter UI)                                   │
└───────────────────────────────────────────────────────────────────────────┘

    👤 User
     │
     ├─ Selects report type: "Inventaire Résumé"
     ├─ Selects format: PDF / Excel / CSV
     └─ Clicks "Exporter" button
          ↓
    📱 BloodBankInventoryPage._exportReport()
     │
     ├─ Formats dates: startDate="2025-10-12", endDate="2025-11-12"
     ├─ Shows loading SnackBar: "Génération du rapport en cours..."
     └─ Calls Controller
          ↓


┌───────────────────────────────────────────────────────────────────────────┐
│  PHASE 2: CONTROLLER LOGIC (Riverpod State Management)                    │
└───────────────────────────────────────────────────────────────────────────┘

    🎮 BloodReportExportController.exportReport()
     │
     ├─ Updates state: isExporting=true, isDownloading=true
     └─ Calls API Service
          ↓


┌───────────────────────────────────────────────────────────────────────────┐
│  PHASE 3: API CALL (HTTP Request)                                         │
└───────────────────────────────────────────────────────────────────────────┘

    🌐 BloodReportsApiService.generateBloodReport()
     │
     ├─ Builds URL: /api/v1/eblood/reports/blood-inventory/generate
     ├─ Query params: {report_type, format, start_date, end_date}
     └─ HTTP GET Request → Backend
          ↓


┌───────────────────────────────────────────────────────────────────────────┐
│  PHASE 4: BACKEND PROCESSING (Python FastAPI)                             │
└───────────────────────────────────────────────────────────────────────────┘

    🐍 blood_reports_inventory_controller.generate_blood_report()
     │
     ├─ [1] Authenticate user (verify_logged_in_user)
     ├─ [2] Get organization_id from JWT token
     ├─ [3] Query MongoDB for blood bags in date range
     │       └─ Collection: STOCK_BLOOD_BAG
     │       └─ Filter: {sys_organization_id, created_at: {$gte, $lte}}
     │
     ├─ [4] Process data by report_type
     │       ├─ inventory_summary → Group by blood type, status, product
     │       ├─ collection_stats → Daily collections, averages
     │       ├─ distribution_stats → Used/transferred bags
     │       └─ expiration_analysis → Expired, expiring soon, good
     │
     ├─ [5] Generate file
     │       └─ Path: {cwd}/static/reports/{report_file_name}
     │       └─ Methods: _generate_pdf_report() / _generate_excel_report() / _generate_csv_report()
     │       └─ Creates actual file on disk
     │
     └─ [6] Return JSON response
          ↓

    📤 Response:
    {
      "success": true,
      "message": "Report generated successfully",
      "data": {
        "file_name": "inventory_summary_20251012_to_20251112.pdf",
        "file_generated": true,          ← ✅ Confirms file created
        "file_size_bytes": 2048,         ← 📊 File size
        "download_url": "/eblood/reports/blood-inventory/download?file_name=...",
        "report_data": { ... }
      }
    }
          ↓


┌───────────────────────────────────────────────────────────────────────────┐
│  PHASE 5: AUTOMATIC DOWNLOAD (Flutter Downloads File)                     │
└───────────────────────────────────────────────────────────────────────────┘

    🔽 BloodReportExportController (continues)
     │
     ├─ Checks: file_generated == true ✅
     ├─ Logs: "📄 File generated: true"
     ├─ Logs: "📁 File name: inventory_summary_20251012_to_20251112.pdf"
     ├─ Logs: "📊 File size: 2.00 KB"
     └─ Auto-triggers: downloadReportFile(downloadUrl)
          ↓

    📥 BloodReportsApiService.downloadReportFile()
     │
     ├─ [1] Gets app documents directory
     │       └─ /data/user/0/com.example.app/app_flutter/
     │
     ├─ [2] Creates Dio instance with 5-minute timeout
     │
     ├─ [3] Builds full URL
     │       └─ http://localhost:4548/eblood/reports/blood-inventory/download?file_name=...
     │
     ├─ [4] Downloads file via Dio
     │       └─ dio.download(fullUrl, filePath, onReceiveProgress: ...)
     │       └─ Progress: "📥 Download progress: 25%... 50%... 100%"
     │
     ├─ [5] Saves to local storage
     │       └─ /app_flutter/inventory_summary_20251012_to_20251112.pdf
     │
     └─ [6] Opens file with Share API
          ↓


┌───────────────────────────────────────────────────────────────────────────┐
│  PHASE 6: FILE SERVING (Backend Serves File)                              │
└───────────────────────────────────────────────────────────────────────────┘

    🐍 blood_reports_inventory_controller.download_blood_report()
     │
     ├─ [1] Gets file_name from query param
     │
     ├─ [2] Validates file_name (no path traversal)
     │       └─ Blocks: "../", "./", "~/"
     │
     ├─ [3] Constructs file path
     │       └─ reports_dir = Path.cwd() / "static" / "reports"
     │       └─ file_path = reports_dir / file_name
     │
     ├─ [4] Checks file exists
     │       └─ if not file_path.exists(): raise HTTPException(404)
     │
     ├─ [5] Determines MIME type
     │       ├─ .pdf → "application/pdf"
     │       ├─ .csv → "text/csv"
     │       └─ .xlsx → "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"
     │
     └─ [6] Returns FileResponse
          └─ Headers: Content-Disposition: attachment; filename="..."
          └─ Streams file to client
          ↓


┌───────────────────────────────────────────────────────────────────────────┐
│  PHASE 7: FILE OPENING (System Opens PDF Viewer)                          │
└───────────────────────────────────────────────────────────────────────────┘

    📄 Share.shareXFiles([XFile(filePath)])
     │
     ├─ iOS: Opens UIActivityViewController
     │       └─ Options: Open in Adobe Acrobat, iBooks, Mail, etc.
     │
     ├─ Android: Opens Intent.ACTION_VIEW
     │       └─ Options: Google PDF Viewer, Adobe Reader, etc.
     │
     └─ User sees PDF opened in viewer
          ↓


┌───────────────────────────────────────────────────────────────────────────┐

                            ERROR HANDLING FLOW

═══════════════════════════════════════════════════════════════════════════

    ❌ IF ERROR OCCURS AT ANY STAGE:
     │
     ├─ Backend Error (Generation fails)
     │   └─ Returns: {success: false, message: "Error generating report"}
     │
     ├─ Network Error (Connection timeout)
     │   └─ Dio throws DioException
     │
     ├─ Download Error (File not found)
     │   └─ 404 response from backend
     │
     └─ File Open Error (No PDF viewer)
         └─ Share API prompts to install app
              ↓

    🎮 Controller updates state:
     │
     ├─ isExporting = false
     ├─ isDownloading = false
     └─ error = "Error message"
          ↓

    📱 UI shows error SnackBar:
        ┌─────────────────────────────────────────────────────────┐
        │ ⚠ Erreur lors du téléchargement: [error details]        │
        └─────────────────────────────────────────────────────────┘
            Background: Red
            Duration: 5 seconds
